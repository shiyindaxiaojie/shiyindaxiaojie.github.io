Tomcat 原理

Tomcat 的架构是怎么样的？

Tomcat = HTTP 服务器（连接器、线程池） + Servlet 容器（请求哪里来就应答到哪里）

Tomcat 主要包含两大组件：

  1. 连接器：处理 Socket 连接，负责网络字节流与 Request 和 Response 对象的转化。
    • Endpoint：实现 TCP/IP 协议
    • Processor：实现 HTTP 协议
    • Adaptor：将协议的请求转化 Servlet 的请求
  2. 容器：加载和管理 Servlet，以及具体处理 Request 请求。
    • Service：包含 Connector 定义端口和 Engine
    • Host：虚拟主机
    • Context：应用上下文
    • Wrapper：Servlet 路径

Netty 可以实现 HTTP 服务器，由于 Tomcat 本身的 NIO 实现性能表现良好，而且 Servlet 规范要求 Http Body 请求需要阻塞的，使用 Netty 优势不大。目前，Spring Boot 2.X 版本集成了 Netty，目的应该是抛弃 Servlet 规范来实现高性能 HTTP 服务。

从 Tomcat 的配置文件 server.xml 可以发现，Tomcat 使用 Service 作为抽象，将 Executor 线程执行器、Connector 请求连接器、Engine 引擎组合在一起，

为了支持多种 I/O 模型和应用层协议，一个容器可能对接多个连接器。Tomcat 定义了 org.apache.coyote.ProtocolHandler 作为通信协议的顶层接口，使用 AbstractEndpoint 通信端口实现 TCP/IP 协议，底层由 Processor 来实现 HTTP 协议，然后通过 Adaptor 适配器将 Tomcat 的 Request 请求转换为 Servlet 容器规范的 ServletRequest 请求，然后将请求发送到容器对应的 Servlet 处理。

目前实现 ProtocolHandler 接口的应用层协议有：

  1. HTTP/1.1：例如 org.apache.coyote.http11.Http11NioProtocol，支持 Range、KeepAlive
  2. HTTP/2.0：例如 org.apache.coyote.http2.Http2Protocol ，使用二进制传输，提高性能
  3. AJP：例如 org.apache.coyote.ajp.AjpNioProtocol，用于和 Web 服务器集成（如 Apache）。

Tomcat 设计了 4 种容器，这些容器使用组合模式实现了 Container 接口，Container 继承了 Lifecycle，表明一个容器具有生命周期的管理功能：

容器按父子的关系从大到小依次为，

  1. Engine/Service：通过多个连接器监听不同的端口
  2. Host：虚拟主机，配置不同的主机、端口、路径
  3. Context：对应 Web 应用的名称，保存上下文关系
