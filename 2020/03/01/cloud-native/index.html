Serverless 架构

什么是 Serverless？

Serverless = Faas（函数即服务） + Baas （后端即服务）

Serverless 使用 Faas 冷启动的特性减少计费，帮助我们省钱和节省运维成本。FaaS 中的冷启动是指从调用函数开始到函数实例准备完成的整个过程。Faas 冷启动平均耗时 100~700 毫秒，对于应用来说这个速度是可接受的。

Service Mesh 架构

什么是 Service Mesh？

Service Mesh = 微服务 Sidecar 化 + 流量/故障/监控/认证

Service Mesh 是一个用来进行请求转发的基础设施层，通常以 Sidecar 的形式部署在微服务内，但对微服务是无侵入式的。Service Mesh 主要提供了流量控制、故障转移、授权认证、日志监控追踪等功能，这些功能在微服务架构表现为通过 API 网关暴露在公网，对部署在内网的微服务统一控制和管理，因此，Service Mesh 和 API 网关从功能上有一定的重叠，但它们的侧重点不一样，Service Mesh 使用过 Sidecar 完全接管发送到应用的请求，部署在应用内，而网关部署在应用外的边界上，对多个应用进行聚合和抽象。

另外，Service Mesh 和 K8s 的关系是 Service Mesh 用于解决服务间网络通信的问题，K8s 解决容器编排和调度的问题，通过调度器管理应用的生命周期，这两者都是构成云原生架构的重要组成部分。

如何理解 Istio ？

Istio = 数据平面（基于 envoy 实现重试和熔断）+ 控制平面（Pilot、Mixer 和 Citadel 实现路由、流量、遥测）

Istio 拦截所有网络流量，并通过 K8s 在每个 pod 中注入智能代理作为 sidecar 来应用一组规则。

Istio 的运行至少需要 8GB 和 4 核心 CPU，使用 Docker Desktop 测试时需要调整下。

云原生架构

云原生架构到底是什么？

云原生架构 = 微服务 + 容器 + 服务网格 + 不可变基础设施 + 声明式 API

云原生架构的理念通过云提供基础设施和各种能力，对中间件下层，分离业务和非业务功能，使得开发人员聚焦于业务开发。

也许，云原生未来的形态会发展成 Mecha 架构。Mecha 架构 = 微逻辑（比微服务更轻量） + Mecha 组件（开箱即用、可配置、声明式API）
