算法复杂度

怎么理解时间复杂度和空间复杂度？

首先，复杂度使用大 O 表示法，假设 n 次循环运算，那么时间复杂度记为 $O(n)$。另外，复杂度一般可以分为最好情况、最快情况、平均情况、均摊。时间复杂度指程序执行 n 次的时间，空间复杂度指程序执行 n 次的空间，时间复杂度体现的是快，空间复杂度的目标是省。一般来说，考虑时间复杂度的场景，除了代码上的优化，还有缓存（空间换时间）、磁盘缓冲、顺序读、预读、网络传输的数据压缩等等，而空间复杂度跟运行环境有关，例如，为了避免 JVM 发生 OOM，首先是减少堆的占用，如果避免不了，那么应该考虑堆外内存，当一个机器的内存不够用，这个时候应该考虑分布式部署了，这里不考虑时间换空间，因为时间的成本比空间要大得多。

排序

冒泡排序（Bubble Sort）

分为有序区和无序区，相邻元素比较并交换，较小的放前面（无序区），较大的放后面（有序区），每一趟循环后最大的在最后面，从代码的实现上，需要 2 轮 n 次循环，时间复杂度仍然为 $O(n^2)$。

选择排序

为了减小冒泡排序的交换次数，引入选择排序，同样分为有序区和无序区，只不过有序区在前面，假设长度为 n，每一趟循环后将第 i 个元素（i = 0; i < n; i++）和剩下的元素逐个比较，最小的放在最前面，代码的实现仍然是 2 轮 n 次循环，所以时间复杂度为 $O(n^2)$。

插入排序

和扑克牌原理类似，每次发完牌，从第 2 个牌开始，将牌往左移动，如果当前的牌比前面还小，就往前移动，直到前面的比这个牌小，就插入，然后后面的牌往右移动一位。可以发现，如果排序的元素本身大部分是有序的，恰好每轮抽出的牌大于前面的值，那么时间复杂度降低到 $O(n)$，所以插入排序要比选择排序好一些。不过，每次插入，后面的元素都要向右移动，这里有一定的性能损耗。
