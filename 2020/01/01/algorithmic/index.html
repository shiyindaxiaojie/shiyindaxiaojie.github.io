算法基础

怎么理解算法复杂度？

首先，复杂度使用大 O 表示法，假设 n 次循环运算，那么时间复杂度记为 $O(n)$。另外，复杂度一般可以分为最好情况、最快情况、平均情况、均摊。时间复杂度指程序执行 n 次的时间，空间复杂度指程序执行 n 次的空间，时间复杂度体现的是快，空间复杂度的目标是省。一般来说，考虑时间复杂度的场景，除了代码上的优化，还有缓存（空间换时间）、磁盘缓冲、顺序读、预读、网络传输的数据压缩等等，而空间复杂度跟运行环境有关，例如，为了避免 JVM 发生 OOM，首先是减少堆的占用，如果避免不了，那么应该考虑堆外内存，当一个机器的内存不够用，这个时候应该考虑分布式部署了，这里不考虑时间换空间，因为时间的成本比空间要大得多。

算法有哪些思路？

  1. 二分查找：前提是有序，在二叉树的场景要避免高度差不能太大
    • 拆半：start + ((end - start) >> 1)
  2. 递归/循环：循环优于递归，可以减少不必要的计算，例如斐波那契数列
    • 获取二叉树的树高/深度
    • 归并排序/快速排序
    • 斐波那契数列，例如跳台阶，不能使用递归产生重复计算，要用循环实现
  3. 回溯/重放：回溯一般使用栈实现 DFS，重放使用队列实现 BFS。
    • DFS 二叉树的遍历，如前序遍历、中序遍历、后序遍历
    • BFS 二叉树的层次遍历
  4. 动态规划：
    • 分治法：快速排序、归并排序
  5. 进制运算：
    • 获取进制位 1 的个数：n &= n-1
    • HashMap 高低位运算哈希：hash = h ^ (h >>> 16) ，其中 h 为 32 位的 int，让高 16 位和低 16 位与运算，使散列跟均匀
  6. 逻辑运算符：
    • 短路与/或：如果不允许使用 if、while 等条件，使用 && 或者 || 也能做到
  7. 归纳分析法：
    • 变态跳台阶：每次跳台阶可以跳 n 阶，这个时候不是斐波那契数列了，而是需要找出规律
  8. 排序/堆：
    • 计数排序：一些无序且分布均匀的整型元素，使用数组排序非常方便，还可以在 $O(n)$ 内找到 top K 任意元素
    • 基数排序：找出区分度高的数字或者字符作为基数来排序，例如号码段，带特定前缀的字符
    • 桶排序：当数值包含小数点时，数值差比较大，不适合用排序，改用区间桶来实现。
    • 二叉堆：通过数组来表示完全二叉树，分为最大堆和最小堆，常用于获取 top K 元素

常用的排序算法有哪些？

  1. 冒泡排序（Bubble Sort）：分为有序区和无序区，相邻元素比较并交换，较小的放前面（无序区），较大的放后面（有序区），每一趟循环后最大的在最后面，从代码的实现上，需要 2 轮 n 次循环，时间复杂度仍然为 $O(n^2)$。
  2. 选择排序（Select Sort）：为了减小冒泡排序的交换次数，引入选择排序，同样分为有序区和无序区，只不过有序区在前面，假设长度为 n，每一趟循环后将第 i 个元素（i = 0; i < n; i++）和剩下的元素逐个比较，最小的放在最前面，代码的实现仍然是 2 轮 n 次循环，所以时间复杂度为 $O(n^2)$。
  3. 插入排序（Insert Sort）：和扑克牌原理类似，每次发完牌，从第 2 个牌开始，将牌往左移动，如果当前的牌比前面还小，就往前移动，直到前面的比这个牌小，就插入，然后后面的牌往右移动一位。可以发现，如果排序的元素本身大部分是有序的，恰好每轮抽出的牌大于前面的值，那么时间复杂度降低到 $O(n)$，所以插入排序要比选择排序好一些。不过，每次插入，后面的元素都要向右移动，这里有一定的性能损耗。
  4. 希尔排序（Shell Sort）：插入排序的优化版
  5. 快速排序（Quick Sort）：自顶向下，每一轮随机取一个数，记作 pivot（例如默认取第一位），然后使用首尾指针双向遍历，左指针 left 从左到右遍历是否小于 pivot，如果大于 pivot 就暂停，右指针 right 从右到左，如果小于 pivot 就暂停，此时交换 left 和 right 的数。
