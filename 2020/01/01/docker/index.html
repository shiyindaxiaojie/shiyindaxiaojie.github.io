如何选择 Docker 的网络模式？

Docker 使用了 Linux 的 Namespaces 技术来进行资源隔离，如 PID Namespace 隔离进程，Mount Namespace 隔离文件系统，Network Namespace 隔离网络。

Docker 支持五种网络模式，刚安装完 Docker 默认创建 bridge、host、none 三种网络，可以通过 docker network ls 查看。

  1. none 模式：完全隔离。只有 local，封闭网络，使用 --network=none 创建隔离的网络环境。
  2. host 模式：共用端口。使用宿主机的 Namespaces，直接与外部通信，使用 --network=host 创建。
  3. bridge 模式：桥接网络。Docker 为每个容器创建一个独立的 Namespace，通过 docker -p 参数映射内外端口。这是容器默认的网络模式，Docker 在安装时会创建一个名为 docker0 的 Linux bridge，在不指定--network 的情况下，创建的容器都会默认挂到 docker0 上面，通过 brctl show 可以查看。
  4. container 模式：绑定其他容器。创建容器时指定 --network=container:其他已存在的容器 共享一个Network Namespace，这个容器本身没有网卡、IP、路由配置。
  5. user-defined 模式：用户自定义模式，可选 bridge、overlay、macvlan。bridge 用于网络桥接，overlay 和 macvlan 用于创建跨主机的网络。

Docker 原理

Docker 是怎么实现的？

Docker 实际上是一个由 Linux Namespace、Linux Cgroups 和 rootfs 三种技术构建出来的进程的隔离环境。

  1. Namespace：使用 clone() 函数系统调用创建一个新进程时，在参数中指定 CLONE_NEWPID 参数，创建的新进程 PID 123 会在一个全新的进程空间看到自己的 PID 为 1，实际上在宿主机的 PID 是 123。 不仅仅是 PID Namespace、Docker 还对文件、网络、用户进行了伪隔离，例如 Mount Namespace 隔离文件目录、Network Namespace 隔离网络、User Namespace 隔离用户。
  2. Cgroups：全称为 Linux Control Group，主要用于限制一个进程组能够使用的资源上限，包括 CPU、内存、磁盘、网络带宽等等。
