Linux 基础

什么是进程、线程、协程？

  1. 进程是运行的程序实例，是资源分配的基本单位，每个进程有自己独占的虚拟空间，因此每次上下文切换需要保存分配的内存资源，开销大
  2. 线程是 CPU 调度的基本单位，属于进程的一部分，一个进程至少有一个主线程，通过 fork() 函数创建子线程，线程在内核和用户态的上下文切换只需要保存栈空间的数据，由于线程共享进程分配的内存资源，所以不需要保存，因此，开销要比进程小得多
  3. 协程在线程的基础进一步划分，在用户态完成操作

进程间如何通信？

  1. 管道：单向管道 |、命名管道 mkfifo test -> 终端 1 输入 echo '233' > test -> 终端 2 输入 cat test 显示结果 233，缺点是阻塞
  2. 消息队列：msgget()、msgsend()、msgrcv()、msgctl()，缺点是队列越大，拷贝时间越长
  3. 共享内存：使用 shmget 创建共享内存，通过 ipcs 可以查看这块内存，进程访问这块内存会先拷贝到自己的虚拟空间的一个位置，让内核分配一个地址，然后对这块内存操作，缺点是多个进程互相访问这块内存，数据不一致
  4. 信号量：由于共享内存互斥问题，使用 PV 原语来操作信号量，实现进程互斥，解决资源竞争的问题，例如，当信号量为 0 时可以访问临界资源 S，那么进程 A 访问 S 时，进行 P 操作将 S = S - 1 = -1，接着进程 B 进来发现 S < 0，只能阻塞等待，当进程 A 执行完这块临界区，就进行 V 操作将 S = S + 1 = 0，此时，进程 B 发现 S = 0 了就可以访问临界资源了。
  5. 信号：kill -l 发送信号或者 CTRL + C 中断，例如 kill -9 pid，指的是发送终止进程的进程给 OS 执行
  6. Socket：本地 Socket 和网络 Socket，主要用于跨服务器间通信

进程的调度算法有哪些？

  1. 先来先服务：FIFO，如果遇到长时间作业，短作业等待时间很长时间才能执行
  2. 短作业优先：长时间作业可能永远得不到执行
  3. 高响应比优先：$响应比 = (等待时间 + 服务时间) / 服务时间$
  4. 时间片轮转算法：固定时间片，会导致短的执行完了空等，长的没执行完就需要上下文切换
  5. 多级反馈队列：多级表示有多个队列，每个队列优先级从高到低，同时优先级越高时间片越短。反馈表示如果有新的进程加入优先级高的队列时，立刻停止当前正在运行的进程，转而去运行优先级高的队列；如果等待的时间太长，就提高优先级，如果运行的时间长，就降低优先级。
