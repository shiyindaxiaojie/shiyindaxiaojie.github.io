Java 笔记
发表于 2020-01-01| 更新于 2020-08-08|编程语言
字数总计:3.5k|阅读时长: 12 分钟
评论数:

Java 是一门面向对象的编程语言，由 Sun 公司推出，后续被 Oracle 公司收购。面向对象具有抽象、封装、继承、多态性等特性，容易维护、复用和扩展，但性能会比面向过程低，因为类调用时需要实例化，开销比较大，同时编译的字节码需要通过 JVM 加载，解释为机器码才放到机器上运行，对于性能要求高的场景，如单片机、嵌入式开发、Linux/Unix 等一般采用面向过程开发。

Java 基础

Java 有哪些特性？

  1. 抽象：使用数据实体表示现实生活的某一事物。
  2. 封装：将内部的数据和方法封装起来，对隐藏信息使用 private，公开接口使用 public。
  3. 继承：复用，抽取共同特性，类只能单继承，但接口可以支持多继承。
  4. 多态：方法的重载、类的继承 @Override，使不同类的对象对同一个消息做出不同的响应。

子类能否继承父类的构造方法？

子类不可以继承父类的构造方法，只可以调用父类的构造方法。子类中所有的构造函数都会默认访问父类中的空参数构造函数，这是因为子类的构造函数内第一行都有默认的 super() 语句。从内存实现或者反射的角度，子类继承所有父类的所有数据域和方法（拥有），但没有不具备对某些私有数据域或私有方法的使用权（调用）。

命名标识符可以包含哪些？

标识符可以包含英文字母、0 到 9 的数字，$ 以及 _，但不能以数字开头，同时标识符不能是 Java 的关键字。按照行业的命名规范，常量全大写，下划线隔开，类名用大驼峰式，变量名和方法名使用小驼峰式。

Java 基本类型有哪些？

基本类型 大小（字节） 默认值 包装类
byte 1 (byte)0 Byte
short 2 (short)0 Short
char 2 \u0000(null) Character
int 4 0 Integer
float 4 0.0f Float
long 8 0L Long
double 8 0.0d Double
boolean 8/32 (int) false Boolean

基本类型在声明时系统会自动给它分配空间，而引用类型声明时只是分配了引用空间，必须通过实例化开辟数据空间之后才可以赋值。例如，数组对象也是引用类型，将数组复制到另外一个数组只是复制了一个引用，对另外一个数组的修改对原来的数组也是可见的。另外，Java 虚拟机没有提供 boolean 值专用的字节码指令，boolean 在编译之后都使用 int 基本类型来代替，占 32 位，而 boolean 数组将会被编码成 byte 数组，每个 boolean 元素占 8 位。

Java 运算符的分类和优先级

  1. 赋值运算符：=
  2. 比较运算符：<、>、==
  3. 算数运算符：除了 +、-、*、/、%，还有 i++ 和 ++i
  4. 逻辑运算符：&& 短路与、|| 短路或、! 非
  5. 移位运算符：用 << 表示二进制位往左移移动 n 位，后面补 0，实际上就是对 2 的取模少算 n 次，即乘以 $2^n$，同理，>> 表示除以 $2^n$。
  6. 位运算符：& 与、| 或、^ 异或
  7. 三元运算符：语法为 条件表达式? 表达式1：表达式2

从性能角度来看，在计算数值时，使用移位运算符、位预算直接操作二进制会更快，因为 CPU 的指令集是针对 1 和 0 操作的，加减乘除的运算需要更多的时钟周期。

对象引用的区别和场景

  1. 强引用：即使内存不足发生 OOM 时不会被回收，通过 new 方式或者反射机制完成分配。
  2. 软引用：每次 GC 时如果内存不足时会被回收，通过 SoftReference 创建，适用于缓存场景。
  3. 弱引用：每次 GC 时不管内存是否不足都被回收，通过 WeakReference 创建。
  4. 虚引用：在被回收之前，会被放入 ReferenceQueue 中，通过 PhantomReference 表示，一般用于 GC 统计，NIO 使用虚引用实现了堆外内存的监视。

分析 String 类的源码

  1. String 使用 final 修饰，不可继承。
  2. String 内部维护了一个 char[] value 数组，但是可以通过反射机制进行破坏，非绝对安全。
  3. String 使用 == 比较的是对象地址，比较内容使用 equals()，时间复杂度为字符串的长度，对于超大字符串比较是否相等，使用对象地址 intern() 来比较会好一些。
  4. String 的 equals() 方法逐个比较字符的内容是否相等，hashCode() 方法逐个字符 * 31 并累加获取哈希值，存在哈希碰撞的可能性。
  5. String 的 intern() 方法尝试从堆中的字符串对象池获取，如果没有，就从堆获取，还是没有，就新建对象，放入字符串对象池，并返回这个地址。
  6. String 的 compareTo() 方法从左到右比较每个字符的大小，如果都相等，就比较字符的长度

理解 equals 和 hashCode 方法

  1. Object 类的 equals() 比较的是对象地址，hashCode() 使用 native 比较哈希值。
  2. String 类在没有覆盖方法的情况下，equals() 相等，hashCode() 一定相等，hashCode() 相等不能保证 equals() 相等，因为 hashCode() 是根据 equals() 的内容来生成哈希，意味着哈希碰撞。

Java 特性

Java 8 有哪些特性？

  1. Optional：解决空指针
  2. FunctionalInterface：函数式接口
  3. 接口新增了 default 修饰符，允许在接口实现默认方法
  4. Lambda 表达式：例如针对函数式接口使用 () -> {} 来简化代码
  5. 方法引用：使用 :: 引用类或者对象的方法
  6. 抽象了 Stream 的 API操作：stream() 串行和 parallelStream() 并行，使用 parallelStream() 需要保证线程安全。

Java 集合

Java 常用的集合类有哪些？场景？

从数据结构分为线性结构和散列结构，Collection 接口是线性结构，子类包含 List 列表、Queue 队列、Set 集合，而 Map 接口属于散列结构。这些数据结构的底层无非就是使用数组或者链表实现存储，使用数组的优势在于查找某个元素的时间复杂度为 $O(1)$，但不适合插入、删除、扩容，使用链表可以使某个元素的插入和删除的时间复杂度为 $O(1)$，但不适合查找某个元素，最多可以优化为通过头尾指针实现双端操作。

列表：元素可以重复，有序，使用数组存储

  1. ArrayList -> List -> Collection：使用数组存储，初始大小默认为 10，每次扩容增大 1.5 倍，非线程安全
  2. Vector -> List -> Collection：同样使用数组存储，和 ArrayList 的区别就是每个方法都是用 synchronized 加锁来保证线程安全，并且每次扩容增大 2 倍
  3. CopyOnWriteList -> List -> Collection：相对 Vector` 的性能优化，读取不加锁，写时加锁再复制数组，不适合大数据量的存储
  4. Stack -> Vector：遵循 LIFO 原则实现了栈的操作，如 push()、pop()、peek() 方法，缺点是每个方法都用 synchronized 加锁来保证线程安全
  5. LinkedList -> List、Deque（Queue）：相对 Stack 的性能优化，支持双端操作

集合：元素不重复，无序/有序

  1. HashSet -> Set -> Collection：使用 HashMap 的 key 存储元素，value 存一个内部的 Object，获取一个元素的复杂度为 $O(1)$。
  2. LinkedHashSet -> Set -> Collection：继承了 HashSet，使用 LinkedHashMap 的 key 存储元素
  3. TreeSet -> Set -> Collection：基于 NavigableMap 和 Comparator 实现有序性

队列：遵循 FIFO 先进先出原则

  1. ArrayBlockingQueue -> Queue -> Collection：有界队列，使用对象数组存储，适用于消息队列 LRU 算法
  2. LinkedBlockingQueue -> Queue -> Collection：无界队列，使用链表存储
  3. PriorityQueue -> Queue -> Collection：优先队列，使用对象数组存储，通过 Comparator 排列优先级
  4. DelayQueue -> Queue -> Collection：延迟队列，使用对象数组存储，通过 Delay 接口实现延时，适用于订单支付倒计时

散列：查找单个元素快，但需要注意哈希碰撞的问题

  1. HashMap -> Map：使用数组存储 Map.Entry 元素，下标为 key 的哈希值，哈希碰撞时转换为链表，碰撞次数达到 8 时转化为红黑树，缩容为 6 时退化为链表，非线程安全
  2. LinkedHashMap -> Map：继承 HashMap，额外维护双向链表，多了 head 头引用、tail 尾引用、before 前一个引用、after 后一个引用
  3. TreeMap -> Map：借助 NavigableMap 组合 Comparator 存储
  4. Hashtable -> Dictionary、Map：和 HashMap 是实现类似，但是每个方法都用 synchronized 加锁来保证线程安全

深入理解 HashMap 的数据结构

  1. 使用数组存储 Map.Entry 元素，下标为 key 的哈希值
  2. 哈希值计算先右移 16 位，再进行与运算
  3. 哈希碰撞时转换为链表，碰撞次数达到 8 时转化为红黑树，缩容为 6 时退化为链表。
  4. 元素个数的初始大小为 16，里面还有个负载因子为 0.75，这是一个经验值，在超过 75% 时，哈希碰撞概率大，性能急剧下降，需要扩容
  5. 非线程安全，使用 ConcurrentHashMap 来保证线程安全

ConcurrentHashMap 有没有坑？

ConcurrentHashMap 在 JDK 1.7 时使用 Segement 分段锁，1.8 后改用 CAS 实现。

了解过哪些锁？

  1. 乐观锁：无锁，CAS 比较并交换，例如 AtomicInteger、AtomicReference
  2. 悲观锁：synchronized 关键字、Lock 接口
  3. 读锁（共享锁）：ReentrantReadWriteLock
  4. 写锁（排它锁）：ReentrantLock、synchronized 关键字
  5. 可重入锁：ReentrantLock、synchronized 关键字
  6. 不可重入锁：同一个线程进入同一个方法不能再次获取锁
  7. 公平锁：ReentrantLock.FairSync -> AQS，排队
  8. 非公平锁：ReentrantLock.NonfairSync -> AQS，在初始化的时候尝试先插队，失败就排队
  9. 偏斜锁 -> 轻量级锁 -> 重量级锁：特指 synchronized 的锁升级，线程 A 进入时，在竞争不激烈的情况下得到偏斜锁（Mark word，缓存），如果后续线程 B 竞争，JVM 撤销偏斜锁，升级为轻量级锁，此时线程 B 自旋获取锁（占用 CPU，避免线程上下文切换），自旋到一定的次数如果还获取不到锁，就退出自旋（让出 CPU），此时升级为重量级锁，所有等待的线程都阻塞。

以下是 MySQL 的锁类型，和 Java 的读写锁是共性的

  1. 元数据锁（MDL）：修改表结构会锁住
  2. 表锁：MyISAM 引擎默认的锁粒度，还有 InnoDB 未使用索引也可能产生表锁
  3. 行级锁：InnoDB 引擎支持
  4. 间隙锁：使用 where a = 1 锁的是记录为 1 的数据行，使用 where a between 1 and 2 锁的是区间
  5. Next Key Lock：间隙锁 + 行级锁，将间隙锁的开区间改为闭区间
  6. 乐观锁：在表结构维护一个版本字段
  7. 悲观锁：显示加锁，读锁 + 写锁
  8. S 锁：select * from t lock in share mode，同时读不冲突，读写会冲突
  9. X 锁：select * from t for update，无论是读还是写都会冲突
  10. 意向锁：分为 IS 锁和 IX 锁，由 MySQL 自动添加，和行级的 S 锁和 X 锁不冲突，只会和表级的 S 锁和 X 锁发生冲突。
