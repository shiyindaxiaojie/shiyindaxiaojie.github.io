深入理解 Buffer Pool

Buffer Pool 是 MySQL 的一个内存组件，里面缓存了磁盘的真实数据，对 MySQL 执行的增删改操作，实际上是先在 Buffer Pool 这里执行修改的。
查看 Buffer Pool 的默认配置参数。

从上图可以看出，Buffer Pool 的大小是通过 innodb_buffer_pool_size 设置的，默认为 134217728 (128M)，一般来说，我们的 MySQL 机器的配置是 16 核 32G，Buffer Pool 128M 是偏小了，可以在 my.cnf 分配为 2G。

按着上面的步骤，安装就完成了。先了解下 sysbench 命令相关的参数。

参数 说明
–time=10 连续访问 10 秒
–threads=10 模拟 10 个线程并发访问
–report-interval=1 每隔 1 秒输出压测的报告
–tables=20 模拟 20 张表
–table_size=1000000 模拟 100W 条数据到模拟表
–db-ps-mode=disable 禁止 PS 模式
oltp_read_write/oltp_read_only/oltp_write_only/… oltp_read_write：测试读写 TPS

oltp_read_only：测试只读性能
oltp_write_only：测试写入性能
oltp_insert：测试插入性能
oltp_delete：测试删除性能
oltp_update_index：测试更新索引字段的性能
oltp_update_non_index：测试更新非索引字段的性能 |
| prepare/run/cleanup | prepare：准备模拟数据
run：开始测试性能
cleanup：清理模拟数据 |
